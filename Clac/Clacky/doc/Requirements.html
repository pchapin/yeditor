Clacky Requirements

Functional Requirements

Clacky shall implement the program described in the Clacky User Manual.

Non-Functional Requirements

Platform

Clacky shall run on the Java Virtual machine and thus target all platforms supporting Java. Clacky assumes the existence of an interactive terminal where the user can enter Clacky command lines and view Clacky's textual output. Clacky shall not have a graphica user interface.

Performance

Clacky shall respond to user commands in a "reasonably" timely manner. The user is expected to understand that computation of difficult results may be time consuming.

Security

Clacky has no special security requirements

User Characteristics

Clacky users are assumed to be comfortable installing Java programs and interacting with text mode (console-style) applications. Clacky users are assumed to have the mathematical background necessary to understand the computed resuls (Clacky is not expected to provide any tutorial information).

Scale

Clacky is a single user system, but it shall make full utilization of the host processor and memory in its computation of user requested results.

Data Formats

Clacky is not currently required to read or write to non-volatile storage. No support for standardized file formats is required. No network communication is required.

Internationalization

Clacky shall be used in an English speaking environment. No support for international character sets or numeric formats is required.
